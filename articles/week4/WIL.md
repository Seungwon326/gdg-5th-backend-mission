# 4주차 WIL

4주차에는 예외처리에 대해 학습하면서 서비스 계층에서 각 객체마다 일일이 넣은 예외처리를 코드를 하나의 객체로 만들어 재사용하면서 코드의 가독성이 좋아졌다. 지난 월요일에 프론트엔드와 API 연결을 진행하면서 예외처리의 중요성을 느꼈는데 이번에 학습하면서 좀 더 자세하게 공부 할 수 있었고 내 코드에 직접 반영해보면서 더 간단하고 가독성 좋은 코드로 수정 할 수 있었다. 4주차에서는 ResponseStatusException을 이용해서 예외처리를 했는데 ResponseStatusException보다 구현 난이도가 있지만 분업화가 더 높은 RestControllerAdvice도 시험 끝나면 적용해봐야겠다.
